# Thinking — 结构化思考系统

> 一份文档骨架。帮你在思考时不被思绪带偏，填完空、有结论、有记录。

## 它不是什么

- 不是一个 App、不是一套软件
- 不需要安装、不需要依赖
- 就是一堆 Markdown 文件

## 它是什么

当你脑子里有个事在转，却说不清自己在想什么的时候——打开它，它会问你几个问题，帮你把散乱的念头压进一个结构里。

## 结构

```
thinking/
├── README.md              ← 你在这里
├── SKILL.md               ← AI 助手的使用说明书
├── ROUTER.md              ← 入口：你在做什么类型的思考？
├── templates/             ← 填空模板（每个不超过 5 个空）
│   ├── 当你要做一个决定时.md              ← 做决定
│   ├── 当你要搞懂一个东西时.md              ← 搞懂一个东西
│   └── 当你有个模糊的想法，说不清楚时.md    ← 想法还很模糊
├── archive/               ← 填完的记录
├── META.md                ← 系统本身的反思日志
└── LAST_SYNC.md           ← 最后一次推送到 GitHub 的时间
```

## 怎么用

### 自己用

打开 `ROUTER.md`，跟着问题走，找到你要填的模板。填完，存到 `archive/`。

### 配合 AI 用

把 `thinking/` 放到你的 AI 助手的 skills 目录下（Claude Code、Cursor 等），AI 会自动加载 `SKILL.md` 的指令。

当你跟 AI 说"我在想一件事"、"帮我理一下思路"、"我有个决定要做"——AI 会引导你完成整个思考。

或者直接：
```
/think                         # 打开路由
/think 当你要做一个决定时        # 直接加载决策模板
/think meta                    # 查看系统状态
```

### AI 的角色

AI 是**引导者**，不是**替代思考者**。它会问你问题、帮你发现盲区、复述你的想法让你确认——但不会替你填模板。

## 核心设计

| 原则 | 说明 |
|------|------|
| 路由不靠记忆 | ROUTER.md 每次打开就在眼前，不需要记 |
| 模板极简 | 每份模板最多 5 个空，超过就是在写文档不是思考 |
| 用完即评 | 每次填完模板顺手评一下：好用 / 一般 / 不好用 |
| 懒改 | 同一个模板两次不好用才改，不折腾 |
| 闭环 | 模板 → 使用 → 评价 → 模板进化（记录在 META.md） |

## 闭环

```
你用模板想了一件事
    ↓
填完、存到 archive/
    ↓
模板末尾勾一下：这次顺吗？
    ├── 顺 → 结束
    ├── 一般 → 没事
    └── 不顺 → 卡在哪了？
              ↓
         攒两次同样的不满 → 改模板
              ↓
         记到 META.md
```
